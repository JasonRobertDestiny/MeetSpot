## 正文

**终结群聊里的无效社交：我开发了一个聚会"公平选址"的 AI Agent**

约几个朋友出去玩，光选地点就聊了二十分钟。
每个人都说"随便"，但真选个地方，总有人要跨半个城，心里其实都不乐意。
这种"三个人迁就一个人"的场景，相信你也遇到过。

为了终结这种群聊里的无效社交，利用课余时间，我做了这个聚会选址 Agent。
[配图：MeetSpot 首页截图]

为什么叫 Agent 而不是工具？
因为它不是让你一步步操作的计算器，而是你告诉它"我们几个人要找个地方聚"，它自己完成整个流程：
解析地址 → 算出公平中心点 → 搜索附近场所 → 综合打分排序 → 给你推荐结果。
你只需要输入位置，剩下的它自己决策。
[配图：地图上显示多个位置 + 汇聚到中心点的截图]

中心点的算法不是简单取平均。
用的是球面几何的地理中心，目的是让所有人的通勤成本降到最低。
不再出现"明明我们三个都在东边，结果选了个西边的地方"这种情况。
[配图：中心点计算示意图]

光定个位置没用，还得有地方可去。
Agent 会扫描中心点附近的店，支持 12 种场景：咖啡馆、餐厅、电影院、KTV、篮球场、健身房、图书馆、博物馆、商场、公园、景点、酒吧。
还能自定义关键词，游泳馆、密室逃脱、剧本杀都能搜。
[配图：场景选择界面截图]

搜到结果后，Agent 会按自己的一套逻辑打分：
- 离中心点近不近
- 评分高不高
- 人气旺不旺
- 符不符合你的需求（安静、有停车位、适合谈事）

最后只推前八个。既然是解决选择困难症，就没必要给太多选项。
[配图：推荐结果页截图，显示评分和卡片]

还加了几个我觉得有意思的功能：
- Agent 会展示它的"思考过程"，让你知道它怎么选的
- 结果页有去各个地点的交通建议，公交地铁打车都有
- 搜不到结果会自动扩大范围，最远能搜 50 公里
- 还有个 AI 客服，不会用可以直接问它
[配图：AI 思考过程 + 交通建议截图]

输入体验也做了优化。
接入了地图 API，支持九十多个高校和地标的简称。输入"北大"就是北京大学，输入"五道口"直接定坐标。
地址输入有自动补全，选了就直接带经纬度，不用手打全称。
[配图：输入框自动补全的截图]

目前是 Demo 版本，部署在 Render 的免费服务上。
不需要注册，不需要登录，点开就能用。
下次群里再有人说"随便"，直接甩这个链接让他填位置。还在迭代中，好用难用都欢迎在评论区告诉我。

传送门：meetspot-irq2.onrender.com

（注：免费服务器如果不活跃会自动休眠，第一次打开如果慢，请多等几秒）

## 为什么是 Agent 而不是工具

| 维度 | 传统工具 | MeetSpot Agent |
|------|----------|----------------|
| 交互模式 | 用户逐步操作 | 输入位置，Agent 自主完成 |
| 决策方式 | 用户决定每一步 | Agent 自主规划执行 |
| 适应性 | 固定流程 | 动态调整（降级搜索、多场景并发） |
| 可解释性 | 无 | 展示 AI 思考过程 |

---

## 解决的核心痛点

| 痛点 | MeetSpot 解决方案 |
|------|------------------|
| 无休止的争论 | 算法定中心点，没得争 |
| "三个人迁就一个人" | 球面几何中心，对所有人公平 |
| 不知道附近有什么 | 12 种场景 + 自定义关键词 |
| 选择困难症 | 综合打分，只推前 8 个 |
| 手打地址麻烦 | 自动补全 + 90+ 简称映射 |
