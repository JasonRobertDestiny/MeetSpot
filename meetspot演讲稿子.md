## **痛点**

这个产品源于我自己的需求

就是有一次朋友来北京找我玩，然后我自己对于北京这个地方也不是很熟悉，平常不怎么去比如咖啡馆、KTV、茶楼这种地方，然后就不知道我们两在哪进行会面，然后在vx群里讨论了几个小时，那么太远不方便，那么环境太差，最终随便选了一个咖啡馆来进行会面

后面我就在想为什么不把我这个需求做成一个Agent呢，因为我自己也是一名vibecoding&agent爱好者，平常也会vibe一些Agent产品，于是我deepresearch做了相关的调研，发现这不只是我的问题，有大约75%的年轻人关注线下活动，同时在X上，有人直接发帖说：需要有人发明一个App，输入两个人的地址，推荐中间见面的地方，小红书也有公司团建选址的需求

但是在高德、白度地图只能搜索我附近有咖啡馆，而搜不了我们中间有什么咖啡馆值得我们去

没有一款产品专注于解决“多人公平聚会”这个问题

这也就是MeetSpot的起源





## **解决方案**

针对这些需求和痛点

我开始撰写PRD，核心功能就是：用户输入多个地址，Agent自主计算球面几何中心点，智能的推荐最优的一些场所

Agent有完整的5步处理流水线：
第一就是地理编码，你输入"北大"，它会自动识别为北京大学，我们内置了90多个智能映射（45所高校简称 + 45个城市地标），不仅服务知识青年，还能自动识别"陆家嘴"、"三里屯"等热门地标

第二，中心计算，用球面几何算法，考虑地球曲率，计算真正公平的中点

第三POI搜索，多场景并发搜索，咖啡馆、餐厅同时异步查找

第四智能排序，多维度评分——包括距离、评价、场景匹配、用户需求

第五，html生成，输出交互式的地图一键导航

传统地图是**被动搜索**——你告诉它搜什么，它就搜什么。

MeetSpot是**主动决策**——你只说"我们在哪"和"想干什么"，它能帮你做出决策。

## 商业模式

商业模式分两块。



先看应用场景：

**B2C**——朋友聚会、约会选址。这是高频场景，每人每月至少2-3次。

**B2B**——企业团建、客户拜访。企业对"公平性"和"可解释性"有更强刚需。

**[指向变现路径]**

变现有三条路：

**第一，B2C增值**：基础功能免费，VIP场景推荐、历史记录收费。

**第二，B2B订阅**：企业版按团队收费，提供管理后台。

**第三，场所合作**：餐厅、咖啡馆付费优先展示。

**[指向愿景]**

更大的想象空间是**平台合作**。

比如Soul——线上匹配成功后，用MeetSpot找一个对双方都公平的见面地点。

## 先对齐一下 Soul 的相关机制

- 奇遇铃 / 灵魂奇遇：本质是基于兴趣、性格等画像的「被动匹配」，铃铛响起=系统认为你们有潜力成为 soulmate，一般会以弹窗/卡片形式提醒你去聊。[4][1]
- LBS 与线下兴趣：奇遇铃升级后加入了线下兴趣爱好与位置维度，会在你附近有高匹配度用户时用铃铛提示，Soul 官方也在强调“同城”“附近”“线下兴趣”的延伸。[2][5][6]
- 位置权限：官方隐私政策里明确提到，开启奇遇铃或位置匹配时，Soul 会使用你的地理位置信息或基于 IP 的模糊位置来做匹配，这为“到达某个区域即触发某种交互”提供了技术条件。[3]

## 你提的合作场景，可以这样抽象

你的设想可以拆成一个完整链路：

1. 线上撮合阶段  
   - 用户在 Soul 内通过「灵魂奇遇 / soulmate」配对成功。  
   - 系统在聊天页或奇遇铃弹层里，以「智能线下见面助手」的形式插入 MeetSpot 小组件（例如：按钮“找个对TA公平的见面点”）。  

2. 线下触发阶段（你说的“达到位置自动触发”）  
   - Soul 端记录双方的城市/大致位置（不必精确到门牌，只需商圈级别）。[3]
   - 双方在 MeetSpot 内选择：  
     - 想见面的大致时间  
     - 可接受的交通方式/最长时间  
     - 期望场景（咖啡聊天、轻食、晚餐、书店等）  

3. MeetSpot 算法与邀请流程  
   - MeetSpot 根据双方位置计算一个「公平中点区」+ 符合 Soul 用户画像的地点清单。  
   - 生成一个“会面计划卡片”：地点候选 + 到达时间估算 + 氛围标签。  
   - 回写到 Soul 聊天窗口，以卡片形式展示，双方可：  
     - 一键投票选择地点  
     - 一键同步到日程 / 置顶在聊天顶部  

4. 到达自动触发  
   - 当任意一方接近目标点一定范围（比如 200–300 米）时，Soul 端可以通过：  
     - App 内系统提示卡片：“你和 TA 快到约会地点啦，要不要发个到达提醒/问候语？”  
     - 触发 MeetSpot 的“见面邀请/签到”组件，比如显示“你们已经在同一个地点，可以解锁一张专属 meet-up 卡片”。  

## 双方合作的互补价值

- 对 Soul 来说：  
  - 现在更强的是“撮合 +聊天”，而线下见面的链路相对留白；接入 MeetSpot，可以把“灵魂奇遇”继续延长到“线下见面的体验设计”，提升成功约会率和用户黏性。[7][8][2]
  - 「奇遇铃 + MeetSpot」可以包装成“灵魂奇遇·线下见面模式”，做活动和故事营销。  

- 对 MeetSpot 来说：  
  - 获取了高匹配度、强情绪动机的用户场景（soulmate / 高匹配度同城），线下见面动机更强，容易验证“选地点”这个刚需。  
  - 通过 Soul 的 LBS 与隐私机制，可以在不直接拿到精确定位的情况下，做「商圈/区域级」的公平选点。
